필요없는 검사 예외 사용은 피해라

핵심: 검사 예외는 편리하지만, 남용하면 코드를 복잡하게 만듬

대안1: 예외 대신 상태를 반환하거나 선택적 값(Optional)을 반환하는 방식 고려 필요
    예) 메서드가 실패했을 때 예외를 던지는 대신, null 이나 비어있는 Optional<T>를 반환하여 호출자가 정상적인 로직으로 처리하도록 유도

대안2: 예외가 발생할 상황을 호출자가 미리 테스트할 수있는 메서드를 제공
    예) 큐에서 원소를 꺼내는 pop() 메서드가 비어있을때 EmptyQueueException이 발생하기 때문에, isEmpty() 메서드를 제공하여 호출자가 먼저 상태를 알 수 있도록 함
