표준 예외를 사용하라

핵심: 자바에서 이미 제공하는 표준 예외를 재활용 하고, 무의미한 예외 클래스를 생성하지 말아라

IllegalArgumentException:
메서드에 전달된 인자의 값이 유효하지 않을 때 사용합니다. 예를 들어, 나이에 음수를 전달하거나, 범위 밖의 숫자를 전달하는 계약 위반 상황에 적합합니다.

IllegalStateException:
메서드 호출 시 객체의 현재 상태가 적절하지 않을 때 사용합니다. 예를 들어, 이미 종료(Close)된 서비스에 요청하거나, 필수적인 초기화가 완료되지 않은 객체에 작업을 시도할 때 발생시킵니다.

NullPointerException:
메서드의 필수 인자나 객체 내부 상태가 null이어서는 안 되는데 null이 전달되었거나 사용되었을 때 발생합니다. (단, 자바 7부터는 Objects.requireNonNull(arg)를 사용하여 명시적으로 검사하고 던지는 것이 좋습니다.)

IndexOutOfBoundsException:
배열, 리스트, 문자열 등 컬렉션에 접근하는 인덱스가 유효한 범위를 벗어났을 때 사용합니다.

UnsupportedOperationException:
객체가 요청된 특정 메서드(작업)를 지원하지 않을 때 사용합니다. 예를 들어, 불변(Immutable) 컬렉션에 add나 remove 같은 변경 작업을 시도할 때 발생시킵니다.

NoSuchElementException:
반복자(Iterator)나 컬렉션(예: 큐, 스택)에서 다음 요소가 존재하지 않는데 요소를 요청했을 때 사용합니다.

TimeoutException :
특정 작업이 지정된 시간(타임아웃) 내에 완료되지 못했을 때 사용합니다. 특히 비동기 처리나 네트워크 통신에서 유용합니다.

NumberFormatException :
문자열을 숫자 타입으로 변환하려 할 때, 해당 문자열의 형식이 유효한 숫자의 형식이 아닐 때 사용합니다. 이는 IllegalArgumentException의 특별한 형태이며, 보통 Integer.parseInt() 같은 변환 메서드에서 발생합니다.
